# N-Reader (NGA 第三方阅读器) 产品需求文档 (PRD)

## 1. 产品概述
- **产品名称**：N-Reader
- **产品定位**：一款专为 NGA 玩家社区打造的第三方移动端 Web 阅读器。
- **核心目标**：解决官方移动端网页版广告多、加载慢、排版拥挤的问题，提供纯净、流畅、高度可定制（字体/行距/深色模式）的阅读体验，并解决跨域和 GBK 编码等技术痛点。

## 2. 核心功能架构
- **板块模块**：支持查看默认板块、获取并展示用户收藏的板块。
- **帖子列表模块**：支持帖子分页加载、下拉刷新、精准的滚动位置记忆（针对安卓优化）。
- **帖子详情模块**：支持 BBCode 解析、楼层展示、点赞/踩、回复功能、帖子收藏。
- **消息中心**：查看收到的回复、@提醒和短消息。
- **个人与设置模块**：Cookie 登录配置、阅读偏好设置（字体大小、行距）、缓存清理。

## 3. 页面设计与实现逻辑

### 3.1 首页 / 板块列表 (Board List)
**实现逻辑**：
- 前端：使用 CSS Grid 布局展示板块卡片。
- 数据获取：调用 `getFavorBoards`，向后端代理发送 `forum_favor2` 请求。
- 状态管理：优先读取内存缓存 `cachedFavorBoards`，若无则发起网络请求。

### 3.2 帖子列表 (Thread List)
**实现逻辑**：
- 数据获取：调用 `getThreadsByBoard` 获取 `thread.php` 数据。
- 分页加载：监听 `onScroll` 事件，当 `scrollHeight - scrollTop <= clientHeight * 1.5` 时触发 `loadMore`。
- **滚动位置恢复 (核心难点)**：
  - 禁用浏览器默认恢复：`window.history.scrollRestoration = 'manual'`。
  - 点击帖子时，将当前 `scrollTop` 和点击的 `thread.id` 存入 `sessionStorage`。
  - 返回列表时，使用 `requestAnimationFrame` 配合重试机制（最多30次，每次50-100ms），优先通过 `document.getElementById('thread-id').scrollIntoView({ block: 'center' })` 进行精准定位，降级方案为恢复 `scrollTop`。

### 3.3 帖子详情 (Thread Detail)
**实现逻辑**：
- 数据获取：调用 `getPostsByThread` 获取 `read.php` 数据。
- **BBCode 解析**：实现自定义 `parseBBCode` 工具函数，利用正则表达式将 `[b]`, `[quote]`, `[img]` 等标签转换为对应的 React 节点（`<strong>`, `<blockquote>`, `<img referrerPolicy="no-referrer">`）。
- 容错处理：使用 `try-catch` 包裹 BBCode 解析逻辑；对 `post.author` 进行空值检查 `(post.author || "未知").toString().charAt(0)`，防止渲染崩溃。
- 回复交互：点击回复弹出底部抽屉（BottomSheet），输入内容后调用 `replyPost` 接口。

### 3.4 个人中心与设置 (Profile)
**实现逻辑**：
- 身份验证：NGA 限制第三方登录，采用手动提取 Cookie (`ngaPassportUid`, `ngaPassportCid`) 并存入 `localStorage` 的方案。
- 全局状态：使用 React Context (`SettingsContext`) 全局管理 `fontSize` 和 `lineHeight`，并在根节点动态应用 Tailwind CSS 类名。

## 4. 后端架构与接口设计 (BFF 层)
由于 NGA 存在严格的跨域限制 (CORS) 且接口返回 GBK 编码数据，必须引入 Node.js (Express) 作为 BFF (Backend For Frontend) 代理层。

**接口定义**：`POST /api/nga`
- **请求参数**：
  - `url`: 目标 NGA 链接
  - `method`: GET 或 POST
  - `uid`, `cid`: 用户凭证
  - `postData`: POST 请求体数据
- **核心处理逻辑**：
  - 请求头伪装：注入标准的 PC 端 User-Agent。
  - Cookie 注入：将传入的 uid 和 cid 拼接为 Cookie 字符串。
  - 编码转换 (请求)：若为 POST 请求，使用 `iconv-lite` 将 `postData` 从 UTF-8 转换为 GBK 编码的 Buffer。
  - 编码转换 (响应)：接收 NGA 的二进制流，使用 `iconv-lite` 将 GBK 解码为 UTF-8 字符串。
  - 脏数据清洗：
    - 移除 NGA 特有的 JS 变量包裹（如 `window.script_muti_get_var_store=`）。
    - 处理非标准的控制字符和 `\xXX` 16进制转义。
    - 使用 JSON5 库解析不规范的 JSON（如键名未加引号）。
    - 若解析失败，使用正则表达式进行深度修复。

## 5. 关键技术难点与解决方案总结
- **跨域与编码**：搭建 Express 代理层，使用 axios 获取 arraybuffer，通过 iconv-lite 进行 GBK 与 UTF-8 的互转。
- **非标准 JSON**：使用 JSON5 库，并配合正则表达式清洗字符串、补全引号。
- **滚动位置丢失**：禁用浏览器默认 scrollRestoration，使用 sessionStorage 记录点击的元素 ID，返回时利用 requestAnimationFrame 和 scrollIntoView 强制居中定位。
- **图片防盗链**：在所有 `<img />` 标签上强制添加 `referrerPolicy="no-referrer"` 属性。
- **渲染崩溃**：增加严格的空值校验 `(post.author || "未知").toString().charAt(0)`，并使用 try-catch 隔离 BBCode 解析错误。
